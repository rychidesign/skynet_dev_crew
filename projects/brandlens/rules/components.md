# Component Rules

## General Patterns
- All components are functional (no class components)
- Use named exports: `export function MetricCard() {}` (not default export)
- Props type defined inline or as separate type above component
- Colocate component + its types in same file
- Extract subcomponents when a component exceeds 100 lines

## shadcn/ui Usage
- Use shadcn/ui as base for all standard UI elements (Button, Card, Dialog, Table, Input, etc.)
- Do NOT edit files in `components/ui/` — these are generated by shadcn CLI
- Customize via Tailwind classes on usage, not by modifying source
- For composite patterns, wrap shadcn components in `components/shared/`

## Styling
- Tailwind CSS only — no inline styles, no CSS modules, no styled-components
- Color tokens: use CSS variables from shadcn theme (`--primary`, `--muted`, `--border`, etc.)
- Custom brand colors defined in `tailwind.config.ts` extending the theme
- Spacing: use Tailwind scale (p-4, gap-6, etc.) — no arbitrary pixel values
- Responsive: mobile-first, use `md:` and `lg:` breakpoints

## Score and Metric Colors
Use consistently across all score displays:
- Critical (0–24): `text-red-500`, `bg-red-50`
- Weak (25–49): `text-orange-500`, `bg-orange-50`
- Moderate (50–74): `text-yellow-600`, `bg-yellow-50`
- Strong (75–89): `text-blue-600`, `bg-blue-50`
- Excellent (90–100): `text-green-600`, `bg-green-50`
Extract as utility: `getScoreColor(score: number) → { text, bg }`

## Charts (Recharts)
- Wrap each chart type in a dedicated component (TrendChart, PlatformBarChart, etc.)
- Use consistent color palette matching brand colors
- Always include: responsive container, tooltip, accessible labels
- No chart in a component that also handles data fetching — separate concerns
- Loading state: skeleton with same dimensions as chart

## Data Fetching Pattern
- Server components fetch data via services in `lib/services/`
- Client components receive data as props or use hooks
- Loading: use Next.js Suspense + skeleton components
- Error: use error boundaries with retry option

## PlanGate Component Pattern
```tsx
<PlanGate requiredPlan="pro" feature="competitor_analysis">
  <CompetitorTable data={competitors} />
</PlanGate>
```
- If plan sufficient: renders children
- If plan insufficient: renders lock overlay with feature name + upgrade CTA
- Never fetch locked data on frontend — backend omits it from response

## Icons
- Use Lucide React exclusively — no other icon libraries
- Import individually: `import { BarChart3, Lock, ArrowUpRight } from "lucide-react"`
- Size: 16px for inline, 20px for buttons, 24px for feature icons
- Stroke width: 1.5 (matching line art aesthetic)
